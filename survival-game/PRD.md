## 一、产品概述

### 1.1 产品定位

《重生之月薪2000》是一款**文字生存模拟游戏**，玩家扮演月薪2000元的打工人，在资源匮乏的环境中做出各种人生选择，体验生存、成长与奋斗的过程。

### 1.2 目标用户

| 用户群体 | 特征 |
|---------|------|
| 年轻打工人 | 共鸣感强，了解职场生存压力 |
| 策略游戏爱好者 | 喜欢资源管理和决策类游戏 |
| 文字游戏玩家 | 注重剧情和沉浸感 |
| 休闲玩家 | 碎片化时间，随时可玩 |

### 1.3 核心价值

- **共鸣**：真实反映打工人的生存困境
- **策略**：每个选择都有代价，考验决策能力
- **成长**：从底层翻身，体验奋斗的成就感
- **随机**：丰富的事件系统，每局体验不同

## 二、游戏机制设计

### 2.1 核心资源系统

玩家需要管理四种核心资源：

| 资源 | 图标 | 说明 | 影响 |
|------|------|------|------|
| 金钱 | 💰 | 主要资源，用于所有消费 | 归零触发破产结局 |
| 健康 | ❤️ | 身体状况，影响工作和生病风险 | 低于20可能生病 |
| 心情 | 😊 | 心理状态，影响工作效率 | 低心情降低收入 |
| 体力 | ⚡ | 行动能力，每项行动消耗 | 不足无法执行行动 |

### 2.2 游戏流程

```
┌─────────────────────────────────────────┐
│              游戏开始                     │
│         初始化角色、资源、属性            │
└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│           每日行动阶段                    │
│  选择：工作/学习/休息/娱乐/兼职/其他      │
└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│           事件触发阶段                    │

                  │

### 2.3 行动系统

#### 日常行动列表

| 行动 | 消耗 | 效果 | 说明 |
|------|------|------|------|
| 正常上班 | 体力-20 | 金钱+100，心情-5 | 基础工作 |
| 加班 | 体力-40，健康-5 | 金钱+150，心情-15 | 透支身体换钱 |
| 请假 | 金钱-50 | 健康+10，体力+30 | 休息恢复 |
| 学习技能 | 体力-30，金钱-30 | 经验+20 | 提升技能，未来加薪 |
| 网吧通宵 | 体力-20，健康-10 | 心情+20，金钱-20 | 放松娱乐 |
| 兼职外卖 | 体力-35 | 金钱+60 | 下班后兼职 |
| 做饭 | 体力-10，金钱-15 | 健康+5，心情+5 | 自己做饭省钱 |
| 外卖 | 金钱-30 | 心情+3 | 贵但方便 |
| 健身房 | 金钱-50，体力-15 | 健康+10，心情+5 | 锻炼身体 |
| 刷手机 | 体力-5 | 心情+10 | 躺平放松 |

#### 特殊行动（解锁条件）

| 行动 | 解锁条件 | 效果 |
|------|----------|------|
| 跳槽 | 技能等级≥2 | 收入提升20%~50% |
| 副业 | 技能等级≥3 | 开启被动收入 |
| 投资 | 存款≥10000 | 风险收益 |
| 创业 | 资金≥50000 | 高风险高回报 |

### 2.4 事件系统

#### 随机事件池

| 事件类型 | 概率 | 示例事件 |
|----------|------|----------|
| **职场事件** | 30% | 老板画饼、同事甩锅、被迫团建、突然加薪 |
| **生活事件** | 25% | 房租涨价、快递丢失、朋友借钱、中彩票 |
| **健康事件** | 15% | 突然生病、牙疼、失眠、体检异常 |
| **机遇事件** | 15% | 面试机会、副业机会、投资机会 |
| **灾难事件** | 10% | 手机坏了、电脑坏了、被骗钱 |
| **幸运事件** | 5% | 年终奖、红包、捡钱 |

    "salary": 2000,
    "day": 1
  }
}
```

### 3.2 月度结算

| 项目 | 金额 | 触发条件 |
|------|------|----------|
| 工资 | +salary | 每月最后一天 |
| 房租 | -800 | 每月最后一天 |
| 水电 | -100 | 每月最后一天 |
| 话费 | -50 | 每月最后一天 |

### 3.3 状态影响

| 条件 | 效果 |
|------|------|
| 健康 < 20 | 工作效率-30%，可能生病 |
| 心情 < 20 | 工作效率-20%，可能离职 |
| 体力 < 10 | 无法执行消耗体力的行动 |
| 金钱 < 0 | 触发借钱/破产事件 |

### 3.4 结局系统

| 结局 | 条件 | 描述 |
|------|------|------|
| 💀 破产 | 金钱连续30天<0 | 被迫回老家，游戏结束 |
| 🏥 生病 | 健康≤0 | 住进医院，游戏结束 |
| 😭 崩溃 | 心情≤0 | 抑郁离职，游戏结束 |
| 🎉 翻身 | 月薪≥10000 | 成功逆袭，通关达成 |
| 💪 小康 | 存款≥50000 | 实现小目标，通关达成 |
| 🏆 人生赢家 | 月薪≥30000，存款≥10万 | 人生巅峰，完美通关 |
| 🛌 躺平 | 连续30天摸鱼 | 选择躺平生活，特殊结局 |

### 4.2 API 设计

#### 核心接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/game/start` | 开始新游戏 |
| GET | `/api/game/status` | 获取当前状态 |
| POST | `/api/game/action` | 执行行动 |
| GET | `/api/game/events` | 获取今日事件 |
| POST | `/api/game/choice` | 做出事件选择 |
| GET | `/api/game/endings` | 获取结局列表 |
| GET | `/api/leaderboard` | 排行榜 |

#### 请求/响应示例

```json
// POST /api/game/start
{
  "player_name": "小明"
}

// Response
{
  "game_id": "uuid-xxxx",
  "player": {
    "name": "小明",
    "money": 2000,
    "health": 80,
    "mood": 60,
    "energy": 100
  },
  "day": 1,
  "message": "欢迎来到《重生之月薪2000》，你是一个刚入职的实习生..."
}

// POST /api/game/action
{
  "game_id": "uuid-xxxx",
  "action": "work_overtime"
}

// Response
{
  "success": true,
  "changes": {
    "money": +150,
    "energy": -40,
    "health": -5,
    "mood": -15
  },
  "message": "加班到深夜，拿了150块加班费，感觉身体被掏空..."
}
```

## 七、风险与应对

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 数值不平衡 | 高 | 中 | 多轮测试调整，Beta测试 |
| 事件内容不足 | 中 | 中 | 持续补充，社区贡献 |
| 玩法单一 | 中 | 高 | 添加成就、排行、随机模式 |
| 内存泄漏 | 低 | 高 | 代码审查，压力测试 |
