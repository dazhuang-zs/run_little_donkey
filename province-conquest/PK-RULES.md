## 项目简介

**问鼎中原** 是一个纯前端实现的中国省份争霸动画项目。34个省级行政区各拥有独立颜色和历史名人，通过六维属性分维度对决，最终统一全国。

### 核心特性
- 34个省级行政区，每个省份拥有独特颜色
- 数百位历史名人，各有六维属性
- 分维度对决机制，公平竞技
- 颜色渐变感染动画，视觉效果震撼
- 并发PK执行，多场对决同时进行
- 右侧实时展示对决详情和名人信息

## PK对决规则

### 1. 省份实力计算

省份的六维实力 = 该省所有名人（原始名人 + 吸收的名人）对应属性的**总和**。

例如：
- 河南省有诸葛亮（智98）+ 岳飞（智78）+ 老子（智100）
- 河南省智谋属性 = 98 + 78 + 100 = 276

### 2. 对决配对规则

每轮开始时：
1. 获取所有**存活**（未被征服）的省份
2. 随机打乱顺序
3. 每个省份优先选择**相邻的存活省份**作为对手
4. 如果没有相邻对手，则随机选择任意其他存活省份
5. 确保每个省份最多参与一场对决
6. 如果只剩一个省份，游戏结束

### 3. 分维度对决

两个省份对决时，**六个维度逐一比较**：

| 维度 | 比较规则 |
|-----|---------|
| 统帅 | 攻方统帅总和 VS 守方统帅总和 |
| 武力 | 攻方武力总和 VS 守方武力总和 |
| 智谋 | 攻方智谋总和 VS 守方智谋总和 |
| 政治 | 攻方政治总和 VS 守方政治总和 |
| 魅力 | 攻方魅力总和 VS 守方魅力总和 |
| 技艺 | 攻方技艺总和 VS 守方技艺总和 |

每个维度的胜负判定：
- **数值高者**赢得该维度
- 如果**数值相等**，攻方胜（进攻优势）

### 4. 胜负判定

统计六个维度的胜负数：

| 情况 | 结果 |
|-----|------|
| 攻方赢4个或更多维度 | 攻方胜 |
| 守方赢4个或更多维度 | 守方胜 |
| 3:3 平局 | 比较**六维总和**，总和高者胜 |
| 总和也相等 | 攻方胜（进攻优势） |

### 5. 征服与合并

对决结束后：
- **胜者**吸收**败者**的所有名人
- **败者**被标记为「已征服」，不再参与后续对决
- 败者省份的**颜色**变为胜者省份的颜色
- 败者的所有名人被转移到胜者的「吸收名人」列表中

### 6. 并发执行

每轮所有对决**同时并发执行**：
- 所有配对同时生成
- 所有对决同时计算结果
- 所有征服动画同时播放
- 右侧同时展示所有对决详情

## 游戏流程

### 完整动画序列

```
1. 初始化
   ├─ 加载中国地图
   ├─ 初始化34个省份数据
   └─ 每个省份显示初始颜色

2. 开场等待（2秒）
   └─ 展示标题「问鼎中原」

3. 主循环（直到只剩一个省份）
   ├─ 3.1 配对阶段
   │   ├─ 获取所有存活省份
   │   ├─ 随机打乱顺序
   │   └─ 两两配对（优先相邻）
   │
   ├─ 3.2 展示阶段
   │   ├─ 右侧展示所有对决列表
   │   ├─ 每个对决展示双方名人（前3位）
   │   ├─ 展示六维对比条
   │   └─ 交战省份金色边框闪烁
   │
   ├─ 3.3 观察等待（2秒）
   │   └─ 观众查看对决详情
   │
   ├─ 3.4 征服动画
   │   ├─ 所有失败省份同时播放颜色渐变
   │   └─ 颜色从败者色渐变为胜者色（1.5秒）
   │
   ├─ 3.5 更新阶段
   │   ├─ 胜者吸收败者名人
   │   ├─ 败者标记为已征服
   │   ├─ 更新右侧名人列表
   │   └─ 添加战斗记录
   │
   └─ 3.6 轮间停顿（0.8秒）
       └─ 准备进入下一轮

4. 胜利动画
   ├─ 全屏浮层显示
   ├─ 展示冠军省份名称
   ├─ 雷达图展示冠军综合属性
   ├─ 滚动展示所有收编的名人
   └─ 粒子特效庆祝
```

## 扩展接口

项目为后续开发预留了完整的扩展接口。

### 1. 新增人物

#### 方式一：直接在数据中添加

在 `js/data.js` 的对应省份 `heroes` 数组中添加：

```javascript
// 在河南省添加新人物
henan: createProvince({
  name: "河南",
  color: "#e74c3c",
  heroes: [
    // 现有名人...
    { 
      name: "新人物名", 
      title: "新称号", 
      stats: { 
        command: 80, 
        martial: 75, 
        wisdom: 90, 
        politics: 85, 
        charisma: 70, 
        craft: 60 
      } 
    }
  ],
  adj: [...]
})
```

#### 方式二：运行时动态添加

使用 `addHeroToProvince()` 函数：

```javascript
addHeroToProvince('henan', {
  name: "新人物名",
  title: "新称号",
  stats: { command: 80, martial: 75, wisdom: 90, politics: 85, charisma: 70, craft: 60 },
  skills: []  // 可选
});
```

### 2. 调整人物属性

直接修改名人 `stats` 对象中的数值即可：

```javascript
// 调整诸葛亮的武力值
{ 
  name: "诸葛亮", 
  title: "卧龙先生", 
  stats: { 
    command: 92, 
    martial: 50,  // 从45调整为50
    wisdom: 98, 
    politics: 96, 
    charisma: 95, 
    craft: 88 
  } 
}
```

无需修改任何其他文件，`getProvinceStats()` 会自动重新计算。

### 3. 新增省份

在 `js/data.js` 中添加：

```javascript
newProvince: createProvince({
  name: "新省份名",
  color: "#颜色代码",
  heroes: [
    { name: "人物名", title: "称号", stats: { ... } }
  ],
  adj: ["相邻省份key1", "相邻省份key2"],
  buffs: []
})
```

同时在 `geoNameMap` 中添加映射：

```javascript
const geoNameMap = {
  // ... 现有映射
  "新省份名": "newProvince"
};
```

## 常见问题

### Q: 如何调整动画速度？

A: 编辑 `js/animation.js` 中的延迟时间：
- `delay(2000)` - 观察等待时间（毫秒）
- `delay(800)` - 轮间停顿时间（毫秒）
- `animationDuration: 1500` - 征服动画时长（毫秒）

### Q: 如何添加更多名人？

A: 参见「扩展接口」章节，支持直接编辑数据或运行时动态添加。

### Q: 游戏如何结束？

A: 当只剩一个存活省份时，触发胜利动画。

### Q: 可以暂停动画吗？

A: 当前版本不支持暂停，动画自动播放直到结束。
