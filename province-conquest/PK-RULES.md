# 问鼎中原 — PK对决说明文档

## 1. 概述

"问鼎中原"是一款基于中国34个省级行政区的策略对战游戏。每个省份拥有若干历史名人，名人具有六维属性值。游戏以回合制进行，每轮所有存活省份并发配对对决，胜者吞并败者的全部名人，败者被淘汰。如此循环直到最终只剩一个省份统一天下，即为"问鼎中原"。

---

## 2. 六维属性系统

### 2.1 属性维度定义

| 属性 | 英文Key | 说明 |
|------|---------|------|
| **统帅** | command | 军事指挥与调度能力，衡量战略统筹、行军布阵的水平 |
| **武力** | martial | 个人战斗力与军事素养，衡量武艺高低、沙场搏杀的能力 |
| **智谋** | wisdom | 智慧与谋略水平，衡量运筹帷幄、诡计奇策的能力 |
| **政治** | politics | 治国理政与外交能力，衡量制度建设、政治手腕的水平 |
| **魅力** | charisma | 人格魅力与号召力，衡量感召人心、聚拢人才的能力 |
| **技艺** | craft | 专业技艺与创造力，衡量文学艺术、科技发明、医术工匠等特殊才能 |

### 2.2 属性值范围

- 所有属性值范围为 **1 ~ 100**（整数）
- 100 代表该维度的历史巅峰水平

### 2.3 属性设计原则

- **特长突出**：每位名人在其最擅长的领域拥有高值（通常 85+）
- **短板明显**：非本职领域会有明显低值，避免"全能战士"
- **总分区间**：单人六维总分通常在 350 ~ 550 之间，极少数顶级人物可达 550
- **历史依据**：属性值参照历史功绩和后世评价设定

## 3. PK对决规则

### 3.1 回合流程

每轮对决按照以下步骤执行：

1. **获取存活省份**：筛选所有未被征服的省份
2. **随机打乱顺序**：使用 Fisher-Yates 算法随机排列存活省份
3. **生成配对**：按打乱后的顺序依次配对（已配对的省份不再参与本轮配对）
4. **执行对决**：每对省份进行六维对决
5. **合并处理**：胜者吸收败者全部名人，败者被标记为已征服
6. **轮次结束**：更新轮次计数，记录对战历史

### 3.2 对手匹配规则

配对遵循以下优先级：

1. **邻接优先**：优先从相邻的存活省份中随机选择对手
2. **远程补充**：若所有相邻省份均已配对或已被征服，则从剩余所有未配对的存活省份中随机选择
3. **跳过处理**：若实在无对手可选（奇数省份时），该省份本轮轮空
4. **保底机制**：若一轮结束后没有产生任何对决且存活省份 > 1，强制取前两个存活省份进行对决

### 3.3 分维度对决规则

两个省份对决时，计算各自所有名人在六维上的属性总和，然后逐维比较：

1. 分别累加双方所有名人的**统帅、武力、智谋、政治、魅力、技艺**总值
2. 六个维度逐一比较，数值高者赢得该维度
3. 单维度数值相同时，**攻方（先手方）获胜**（进攻优势）
4. 统计双方赢得的维度数

### 3.4 胜负判定

- **维度多者胜**：赢得维度数多的一方获胜（如 4:2、5:1、6:0）
- **平局处理（3:3）**：
  - 比较双方六维**总和**（所有维度数值之和）
  - 总和高者胜
  - 总和也相同时，**攻方（先手方）获胜**（进攻优势）

### 3.5 征服后合并规则

- 胜方**吸收败方的所有名人**（包括败方此前已吸收的名人）
- 败方被标记为"已征服"，记录征服者
- 败方**不再参与后续任何对决**
- 胜方后续对决时，使用原始名人 + 所有吸收名人的合计属性

### 3.6 失败省份处理

- 被征服的省份退出游戏，其领土归属征服者
- 若后续需要查找某省的统治者，会递归追溯征服链，找到最终的统治者
- 被征服省份的邻接关系仍然保留（用于地图显示和征服链计算）

## 6. 扩展说明

### 6.1 如何新增人物

在 `js/data.js` 中找到对应省份的 `heroes` 数组，按以下格式添加：

```javascript
{ 
  name: "人物名", 
  title: "称号", 
  stats: { 
    command: 80,   // 统帅 (1-100)
    martial: 75,   // 武力 (1-100)
    wisdom: 90,    // 智谋 (1-100)
    politics: 85,  // 政治 (1-100)
    charisma: 88,  // 魅力 (1-100)
    craft: 70      // 技艺 (1-100)
  } 
}
```

也可以使用扩展接口函数动态添加：

```javascript
addHeroToProvince('henan', {
  name: "新人物",
  title: "称号",
  stats: { command: 80, martial: 75, wisdom: 90, politics: 85, charisma: 88, craft: 70 }
});
```

### 6.2 如何调整属性值

直接在 `js/data.js` 中修改对应人物的 `stats` 对象即可。调整时建议遵循以下原则：

- 保持单维度值在 1-100 范围内
- 特长维度建议 85+，短板维度建议 50 以下
- 单人六维总分建议控制在 350-550 之间
- 参照同省其他人物保持相对平衡

### 6.3 技能系统预留接口

`data.js` 中已预留技能系统扩展接口：

```javascript
// 技能类型枚举
const SKILL_TYPES = {
  PASSIVE: 'passive',    // 被动技能 — 自动生效
  ACTIVE: 'active'       // 主动技能 — 需要触发
};

// 人物对象中的技能字段
{
  skills: [],  // 预留：技能列表
  id: null     // 预留：唯一ID（用于技能引用）
}
```

后续可在 `skills` 数组中添加技能对象，实现属性加成、特殊效果等功能。`createHero()` 工厂函数已支持 `skills` 和 `id` 字段的传入。

### 6.4 Buffs 系统预留说明

每个省份对象中已预留 `buffs` 字段：

```javascript
{
  buffs: []  // 预留：省份Buff列表
}
```

后续可实现省份级别的增益效果，例如：
- **地形Buff**：特定省份在防守时获得属性加成
- **文化Buff**：名人数量达到阈值时触发额外加成
- **士气Buff**：连胜后获得临时属性提升

`createProvince()` 工厂函数已支持 `buffs` 字段的传入和初始化。
