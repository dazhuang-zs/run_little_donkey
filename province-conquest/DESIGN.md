# 中国各省历史人物争霸 - 项目设计文档

## 一、项目概述

**项目名称**：《问鼎中原 - 各省争霸统一天下》

**形式**：可播放的Web动画（支持录屏导出视频）

**核心玩法**：从陕西省（秦国故地）起步，逐步征服相邻省份，最终统一中国

## 三、征服规则设计

### 3.1 基础规则

1. **起点**：陕西省（秦始皇阵营）
2. **扩张方式**：只能征服**相邻省份**
3. **胜负判定**：战力值对比 + 随机加成（±10%）
4. **征服收益**：占领省份后，获得对方代表人物加入阵营

### 3.2 对决公式

```
攻击方战力 = 秦始皇战力 + 已征服省份战力总和 × 0.3
防御方战力 = 省份代表人物战力 × 1.2（主场优势）
最终战力 = 基础战力 × random(0.9, 1.1)

胜负：攻击方最终战力 > 防御方最终战力 → 征服成功
```

### 3.3 征服路径设计

**第一阶段：西北统一**
陕西 → 甘肃 → 宁夏 → 青海 → 新疆

**第二阶段：南下**
陕西 → 四川 → 云南 → 贵州 → 广西 → 广东 → 海南

**第三阶段：东进**
陕西 → 河南 → 湖北 → 湖南 → 江西 → 福建 → 台湾

**第四阶段：北上**
河南 → 河北 → 山东 → 江苏 → 浙江 → 上海
河北 → 北京 → 天津 → 辽宁 → 吉林 → 黑龙江

**第五阶段：统一**
山西 → 内蒙古
安徽补充

## 四、技术实现方案

### 4.1 技术栈选择

| 组件 | 技术选型 | 理由 |
|------|----------|------|
| 地图可视化 | ECharts 5 | 中国地图支持好，动画效果丰富 |
| 地图数据 | 阿里DataV GeoJSON | 免费、准确的省界数据 |
| 动画控制 | CSS3 + JavaScript | 流畅的过渡效果 |
| 人物展示 | 自定义SVG卡片 | 可自定义样式 |
| 录屏导出 | 浏览器MediaRecorder API | 纯前端实现 |

### 4.2 文件结构

```
province-conquest/
├── index.html          # 主页面
├── css/
│   └── style.css      # 样式文件
├── js/
│   ├── main.js        # 主程序入口
│   ├── map.js         # 地图相关逻辑
│   ├── battle.js      # 对决计算
│   ├── animation.js   # 动画控制
│   └── data.js        # 省份数据
├── assets/
│   ├── china.json     # 中国地图GeoJSON
│   └── portraits/     # 人物头像（可选）
└── DESIGN.md          # 设计文档
```

### 4.3 动画效果设计

1. **开场动画**（0-5秒）
   - 地图从空白渐现
   - 各省标上代表人物名字
   - 秦始皇图标闪烁

2. **征服动画**（每场对决约8秒）
   - 攻击方省份高亮
   - 箭头指向目标省份
   - 双方人物卡片弹出
   - 对决特效（战力值滚动比较）
   - 胜负结果展示
   - 省份颜色变更

3. **收尾动画**（5秒）
   - 全地图统一颜色
   - 所有代表人物展示
   - "天下一统"标题
