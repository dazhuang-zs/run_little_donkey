## GitHub Copilot（国际最流行）

### 基本介绍

| 项目 | 内容 |
|------|------|
| **开发商** | GitHub + OpenAI |
| **基于模型** | GPT-4（代码专用微调版）|
| **支持 IDE** | VS Code、JetBrains 全家桶、Neovim |
| **主要功能** | 代码补全、代码解释、Bug 修复、生成测试 |

### 安装步骤

**Step 1：安装 VS Code**

```
官网下载：https://code.visualstudio.com/
```

**Step 2：安装 GitHub Copilot 插件**

```
1. 打开 VS Code
2. 按 Ctrl+P（Mac: Cmd+P）打开命令面板
3. 输入：ext install GitHub.copilot
4. 点击安装
```

**Step 3：登录 GitHub 账号并激活**

```
1. VS Code 左下角点击登录图标
2. 授权 GitHub 账号
3. 首次使用免费 60 天试用期
4. 学生可以申请免费版（需要教育邮箱验证）
```

### 核心功能详解

#### 功能一：行内补全（Inline Completion）

**触发方式：**
- 自动触发：写代码时实时补全
- 手动触发：按 `Alt + \`（Windows/Linux）或 `Option + \`（Mac）

**示例：**
```python
# 输入：
def calculate_fibonacci(n):
    if n <= 1:
        return n
    # Copilot 自动补全：
    else:
        return calculate_fibonacci(n-1) + calculate_fibonacci(n-2)

# 输入：
class User:
    def __init__(self, name, email):
    # Copilot 自动补全：
        self.name = name
        self.email = email

# 输入：
# 连接数据库的函数
def connect_db(host, port, user, password):
    import pymysql
    # Copilot 自动补全：
    return pymysql.connect(host=host, port=port,
                           user=user, password=password)
```

#### 功能二：对话问答（Chat）

**打开方式：**
- 快捷键：`Ctrl + Shift + I`（Windows/Linux）或 `Cmd + Shift + I`（Mac）
- 或点击左侧边栏的 GitHub 图标

**常用命令：**

| 命令 | 功能 | 示例 |
|------|------|------|
| `/explain` | 解释代码 | 选中代码后输入 `/explain` |
| `/fix` | 修复 Bug | 输入 `/fix 修复这个函数的错误` |
| `/tests` | 生成测试 | 输入 `/tests 为这个函数生成单元测试` |
| `/refactor` | 重构代码 | 输入 `/refactor 优化这段代码` |
| `/docs` | 生成文档 | 输入 `/docs 生成这个函数的文档` |

**实战示例：**

```
用户：
/explain 这段代码是做什么的

Copilot 回答：
这是一个用户认证函数，它做了以下事情：
1. 接收用户名和密码作为参数
2. 验证输入不为空
3. 从数据库查询用户
4. 使用 bcrypt 验证密码
5. 返回 JWT token 或认证失败信息

复杂度：O(n)，n 为数据库查询时间
潜在问题：如果数据库连接失败会抛出异常
```

#### 功能三：生成测试（Tests）

```
1. 选中要测试的函数
2. 按 Ctrl + Shift + I 打开 Chat
3. 输入：/tests
4. Copilot 自动生成测试用例
5. 可选择 pytest/unittest 等框架
```

## Cursor（新生代神器）

### 基本介绍

| 项目 | 内容 |
|------|------|
| **开发商** | Anysphere（YC 孵化）|
| **基于模型** | GPT-4、Claude 3.5 Sonnet（可选）|
| **支持系统** | Windows、Mac、Linux |
| **主要特点** | AI 原生 IDE、团队协作、内置 LLM |

### 安装步骤

```
1. 官网下载：https://cursor.com/
2. 安装后注册登录
3. Settings → Models 选择使用的模型
4. 免费版每天 100 次 Claude 请求，GPT-4 需要付费
```

### 核心功能详解

#### 功能一：AI 对话面板（CMD + K）

**快捷键：** `Cmd + K`（Mac）或 `Ctrl + K`（Windows）

**使用方式：**
- 选中代码后按 `Cmd + K`，可以直接编辑选中代码
- 不选中时，询问后会生成新代码

**示例：**

```
用户输入 CMD + K：
"把这个函数改成支持异步操作"

Cursor 输出：
→ 自动重写函数，添加 async/await
→ 更新函数签名
→ 修改调用处
```

#### 功能二：Composer（多文件生成）

**打开方式：** `Cmd + I` 打开 Composer 面板

**功能：**
- 输入需求描述
- Cursor 自动生成多个相关文件
- 支持预览和修改
- 直接写入项目

**示例需求：**
```
"用 Python + FastAPI 写一个用户注册登录的 API，
包含：注册接口、登录接口、JWT 认证、数据库连接"
```

**Cursor 自动生成：**
```
models/user.py      → 用户模型定义
routers/auth.py     → 认证路由
schemas/user.py     → Pydantic 模型
database.py         → 数据库连接
main.py             → FastAPI 主文件
requirements.txt    → 依赖文件
```

#### 功能三：代码解释与调试

**使用方式：**
1. 选中代码
2. 右键 → "Explain Code"
3. 或在 AI 对话中直接问

**特点：**
- 中文友好
- 解释详细，适合学习
- 可以追问细节

#### 功能四：自动 Bug 修复

```
1. 代码有红色波浪线（语法错误）
2. 按 Cmd + L 打开 AI 面板
3. 输入："修复这个错误"
4. Cursor 给出修复方案和解释
```

| **模型** | 智谱 GLM、Qwen、DeepSeek、OpenAI 等 |
| **适合场景** | 国内开发者调用大模型 API |

### 注册与使用

**Step 1：注册**

```
官网：https://siliconflow.cn/
1. 用手机号注册
2. 新用户送 14 元体验金
3. 实名认证后可获得更多额度
```

**Step 2：获取 API Key**

```
1. 登录后进入控制台
2. 点击左侧"API 密钥"
3. 点击"新建密钥"
4. 复制保存密钥（sk-xxx 格式）
```

**Step 3：调用 API**

```python
# 安装依赖
pip install openai

# 调用示例（以智谱 GLM-4 为例）
from openai import OpenAI

client = OpenAI(
    api_key="你的API密钥",
    base_url="https://api.siliconflow.cn/v1"
)

response = client.chat.completions.create(
    model="THUDM/glm-4-flash",  # 便宜快速
    messages=[
        {"role": "system", "content": "你是一个Python编程助手"},
        {"role": "user", "content": "写一个快速排序算法"}
    ],
    temperature=0.7
)

print(response.choices[0].message.content)
```

## 面试高频问题汇总

### 问题一：你用过哪些 AI 编程工具？用过哪些？

**参考回答：**

> "我用过 GitHub Copilot、Cursor、通义灵码和硅基流动 API。
> 
> Copilot 用得最多，主要用于代码补全和 Bug 修复。它的补全速度最快，准确率很高。
> 
> Cursor 用于独立做小项目，它的多文件生成功能很强，能一次性生成整个项目的框架。
> 
> 通义灵码主要在阿里云服务器开发时用，免费且对中文注释支持好。
> 
> 硅基流动主要用于调用国内大模型 API，价格便宜，接入简单。"

---

### 问题二：AI 编程工具能完全替代程序员吗？

**参考回答：**

> "不能完全替代，但能大幅提升效率。
> 
> AI 擅长：
> - 重复性高的代码（CRUD、模板代码）
> - 代码补全和片段生成
> - Bug 修复和代码优化
> - 解释和学习新代码
> 
> AI 不擅长：
> - 系统架构设计
> - 理解复杂业务逻辑
> - 处理模糊的需求
> - 创造性解决问题的方案
> 
> 现在的定位应该是'高级助手'，帮我处理重复工作，我负责决策和架构。"

---

### 问题三：AI 生成代码有什么风险？怎么规避？

**参考回答：**

> "主要有三类风险：
> 
> **1. 安全隐患**
> AI 可能生成有安全漏洞的代码，比如 SQL 注入、XSS 等。
> 规避：使用安全扫描工具（如 SonarQube），AI 生成的代码必须人工审查。
> 
> **2. 逻辑错误**
> AI 对复杂业务逻辑理解有限，可能生成逻辑错误的代码。
> 规避：AI 代码必须有测试覆盖，不能直接用于生产。
> 
> **3. 依赖版本问题**
> AI 生成的代码可能使用过时库或不兼容版本。
> 规避：检查依赖版本，优先使用 LTS 版本。
> 
> 我的习惯是：AI 生成 + 人工审查 + 单元测试 + Code Review。"

---

> | 代码补全 | 更准确 | 够用 |
> | 中文注释补全 | 弱 | 强 |
> 
> 我的选择：日常开发用 Copilot（补全质量高），国内项目用通义灵码（免费且中文好）。"

### 问题五：你怎么用 AI 提升编程效率？

**参考回答：**

> "我主要在三个方面用 AI 提升效率：
> 
> **1. 编码阶段**
> - 用 Copilot 做代码补全，减少打字量
> - 遇到不熟悉的 API，让 AI 先写一个示例
> - 用 Cursor 的多文件生成快速搭框架

> **2. 代码审查阶段**
> - 写完代码后让 AI 检查潜在 Bug
> - AI 生成单元测试，提升测试覆盖率
> - AI 检查代码规范问题
> 
> **3. 学习阶段**
> - 看开源项目时用 AI 解释不熟悉的代码
> - 学习新框架时让 AI 写入门示例
> - 让 AI 推荐最佳实践
> 
> 粗略估计，AI 帮我节省了 30-40% 的编码时间。"

### 问题六：你了解 RAG 吗？有没有用过？

**参考回答：**

> "了解，并且用过。
> 
> RAG = Retrieval-Augmented Generation（检索增强生成）
> 
> **原理：**
> 1. 把知识库文档切分成小块，向量化存储
> 2. 用户提问时，从知识库检索相关内容
> 3. 把检索结果 + 问题一起发给大模型
> 4. 大模型基于检索结果生成准确答案
> 
> **我用过的方式：**
> - 用 LangChain + 硅基流动 API 搭建过内部知识库
> - 用扣子 Coze 的知识库功能创建过客服 Bot
> 
> **为什么用 RAG：**
> 大模型有知识截止日期，且可能"幻觉"（编造答案）。RAG 让模型能访问最新、最准确的知识。"

### 问题七：AI 编程工具未来会怎么发展？

**参考回答：**

> "我认为会朝三个方向发展：
> 
> **1. 更深度集成**
> 从"辅助补全"到"主导开发"——AI 负责大部分编码，人负责审核和决策。
> 
> **2. Agent 化**
> AI 不仅写代码，还能自主规划任务、调用工具、执行测试、修复 Bug。
> 比如 Cursor Agent、MarsCode Agent 已经能做完整的开发任务。
> 
> **3. 垂直领域深化**
> 针对不同行业（金融、医疗、法律）推出专用模型，理解领域术语和最佳实践。
> 
> **4. 安全合规**
> 企业级 AI 编程工具会加强安全扫描、合规检查、代码溯源等能力。
> 
> 对程序员来说，学会和 AI 协作比单纯写代码更重要。"
