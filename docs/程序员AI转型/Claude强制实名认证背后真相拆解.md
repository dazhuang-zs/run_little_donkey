# Claude 强制实名认证背后：一篇让你看懂本质的文章

> 4月14日，Anthropic 突然宣布 Claude 平台推行 KYC 身份验证。消息一出，全网炸锅。国内用户更是首当其冲。这篇文章不跟风站队，只把事实摆出来，你自己判断。

---

## 一、先说发生了什么

4月14日，Anthropic 官方发布公告，宣布 Claude 平台启动**强制身份验证（KYC）**。

具体要求：

1. **证件要求**：政府签发的实体证件原件（护照、驾照、身份证）
2. **不接受**：复印件、扫描件、数字证件、电子证件、临时身份证
3. **验证方式**：手持证件原件 + 人脸实时自拍（由第三方供应商 Persona 执行）
4. **耗时**：通常不超过 5 分钟

触发条件包括：
- 访问特定功能
- 触发平台风控（完整性检查）
- 安全和合规措施相关场景

**官方明确：即便完成验证，账号仍可能被禁用。禁用理由包括：**
- 重复违反使用政策
- 从不支持的地区创建账户
- 违反服务条款
- 未满 18 岁使用

---

## 二、国内用户为什么炸了？

### 核心问题：国内身份证能否通过验证？

目前 Claude 官方对中国大陆居民身份证是否在可接受证件列表中，**表述模糊**。

更关键的是：公告明确提到"从不支持的地区注册，账号直接封"。

**实际案例**：
- 一位在海外的中国用户，用国内手机号注册，已触发验证但无法通过
- 一位 15 岁开发者账号被封，原因正是"未满 18 岁"
- 接码平台批量注册的账号已成片被封

### 被封的账号特征

| 特征 | 说明 |
|------|------|
| 接码平台注册 | 一个手机号注册多账号 |
| API 滥用 | 批量请求、爬虫 |
| 商业转售 | 二手平台倒卖账号 |
| 非 Anthropic 战略市场 | 中国大陆等地区用户 |

### Anthropic 的官方理由

```
防止滥用
  ↓
落实平台政策
  ↓
履行法律合规义务（KYC / AML 要求）
```

这个理由表面上说得通。但市场解读出了更多意味深长的信号。

---

## 三、被忽视的关键细节

### 1. KYC 执行方是 Persona

Anthropic 把身份验证外包给了 **Persona** 这家第三方公司。

Persona 是专门做身份验证的 KYC 服务商，之前服务过 Coinbase、Robinhood 等平台。

问题在于：Persona 接触了用户的证件照片和人脸数据。**这些数据会不会被 Anthropic 用于其他目的？**

公告说"仅用于确认身份，不用于其他目的"。但这个承诺在法律上有多大约束力，目前没有明确说法。

### 2. 中国用户是"不支持的地区"

这是最致命的一条。

Claude 官方从未正式进入中国市场。国内的 API 调用和网页使用，本质上一直是"灰色地带"。

这次 KYC，很可能是 Anthropic 的一次**精准清场**：用合规名义，把非战略市场的用户批量过滤掉。

### 3. 已有用户被封，且无法申诉

多位用户反映：账号被封后，申诉渠道几乎无效。Claude 的封号申诉流程远不如 ChatGPT 完善。

---

## 四、Anthropic 到底在下什么棋？

市场上有几种主流分析：

### 分析一：合规压力（官方说法）

欧洲市场（GDPR）、北美市场都有 KYC 要求。Anthropic 上市在即（年化收入 ARR 已超 300 亿美元，首超 OpenAI），需要补合规短板。

### 分析二：封杀滥用（合理推断）

Claude 一直是"滥用重灾区"：
- 接码平台批量注册
- API 被用于爬虫和批量请求
- 二手平台倒卖账号

KYC 大幅提高了滥用成本。

### 分析三：战略用户分层（阴谋论但值得思考）

Anthropic 的核心收入来自企业客户（Claude Enterprise）和 API 商业调用。

个人用户、薅羊毛党、非核心市场的使用，对 ARR 贡献有限，但带来了合规风险。

**KYC = 精准淘汰低价值高风险用户。**

## 五、国内用户现在怎么办？

### 方案一：API Key（推荐）

如果你用的是 **Claude API Key**（Claude Code、第三方工具等），**目前不受 KYC 影响**。

API 认证走的是另一套系统，用的是 API Key，不是网页账号。

```
✅ Claude Code + API Key → 不受影响
❌ Claude.ai 网页版 → 可能触发验证
❌ Claude Max 订阅 → 可能触发验证
```

### 方案二：换用国产大模型

KYC 事件后，国产大模型（DeepSeek、通义千问、文心一言、智谱GLM）成了更多国内用户的替代选择。

这波操作客观上加速了国产大模型的普及。

### 方案三：等待

KYC 目前只对"部分用户"弹出，不是所有人都会触发。如果你暂时没遇到验证提示，可以先正常使用。

---

## 六、这对中国 AI 生态意味着什么？

Claude 的 KYC 事件，客观上加速了几个趋势：

**1. 国产大模型崛起**
KYC 逼走了一批"灰色用户"，直接利好 DeepSeek、通义千问、智谱等国产模型。

**2. API 代理和镜像站的需求上升**
国内用户访问 Claude API 的成本和门槛都在上升。

**3. 合规 AI 工具成新赛道**
未来 AI 工具出海，合规会是标配而不是加分项。

**4. Anthropic 的企业市场更稳了**
清场个人用户后，Anthropic 的 ARR 数据会更漂亮，但用户规模的增长会放缓。

## 七、一句话总结

```
Claude 的 KYC，本质是：
合规要求 + 滥用治理 + 战略用户分层
= 精准淘汰非核心市场用户

对国内用户的影响：
API Key 方式 → 不受影响
网页版/Max → 风险上升

最坏情况：国内用户可能需要换用国产大模型。
```
