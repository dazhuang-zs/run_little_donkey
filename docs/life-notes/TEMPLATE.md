# 笔记模板

> 每条笔记的标准格式，保证产出质量和一致性。

---

## 模板结构

```markdown
# 标题（可选，抖音文案里一般不需要）

> 开场金句（一句话抓住注意力，前3秒决定生死）

正文内容（100-300字，分段清晰）

结尾金句（收束情绪，留下共鸣点）

---
## 发布信息
- BGM推荐：xxx（情绪匹配）
- 发布时间建议：xxx（情绪类型→最佳时段）
- 预期互动点：xxx（评论区容易讨论什么）
- 标签：#xxx #xxx #xxx
```

---

## 开场金句公式

| 类型 | 公式 | 示例 |
|-----|------|-----|
| **冲突型** | "X的人，都Y" | "深夜写代码的人，都有故事" |
| **悬念型** | "X，你可能不知道" | "程序员焦虑，你可能不知道的秘密" |
| **共鸣型** | "当你X的时候" | "当你凌晨3点还在改bug" |
| **反转型** | "大家都说X，其实" | "大家都说35岁危机，其实..." |
| **画面型** | 时间+地点+动作 | "凌晨3点，写完最后一个bug" |

---

## 正文节奏控制

抖音文字笔记的有效节奏：

```
开场金句（1句）
→ 铺垫背景（1-2句）
→ 核心故事/观点（3-5句）
→ 情绪转折（1句）
→ 结尾金句（1句）
```

总字数控制在 100-300 字之间。

---

## 情绪标签与BGM匹配

| 情绪 | BGM风格 | 最佳发布时段 |
|-----|---------|------------|
| 焦虑 | 低沉、压抑 | 深夜 21:00-23:00 |
| 治愈 | 温暖、轻柔 | 早上 7:00-9:00 |
| 迷茫 | 中性、思考 | 下午 15:00-17:00 |
| 自信 | 节奏感、有力 | 中午 12:00-14:00 |
| 幽默 | 欢快、反差 | 晚上 19:00-21:00 |

---

## 标签规则

抖音标签写法：

```markdown
#程序员的日常        ← 身份标签，圈定人群
#深夜写代码          ← 场景标签，引发共鸣
#焦虑是常态          ← 情绪标签，引发讨论
#35岁危机            ← 热点标签，蹭流量
```

每条笔记 2-4 个标签，不要太多。

## 实例（完整模板应用）

```markdown
> 深夜写代码的人，都有故事。

凌晨3点，改完最后一个bug。

编译通过了。

那一刻，屏幕上的绿色进度条，
比什么都好看。

不是因为代码有多难，
而是因为坚持了很久。

很多人问我，为什么还在写代码。
我说，写代码的时候，
不用说话，不用解释，
只要专注一件事就好。

在这个嘈杂的世界里，
专注，是一种奢侈。

---
## 发布信息
- BGM推荐：安静轻音乐（钢琴版）
- 发布时间建议：深夜 21:00-23:00
- 预期互动点：评论区分享深夜加班经历
- 标签：#程序员的日常 #深夜写代码 #专注的力量
```

## 检查清单

发布前检查：

- [ ] 开场有金句（不是普通开头）
- [ ] 正文分段清晰（每段不超过50字）
- [ ] 结尾有收束情绪的金句
- [ ] BGM情绪匹配
- [ ] 发布时段合理
- [ ] 标签2-4个

全部通过 → 发布。
