# 脚本策划：拍之前先想清楚要剪什么

## 一、为什么99%的新手跳过了脚本这一步

你一定听过这个建议："拍之前先写脚本。"你也一定跳过了它。

不是因为你懒，是因为写脚本这件事本身违反了新手对"创作"的直觉。

大多数人是这样进入视频创作的：看了一个很酷的视频，心里冒出"我也能做"的念头，拿起手机就开始拍。这个过程充满了即兴和冲动，而"写脚本"听起来像是把一件热血的事情变成了写作业。

但真正的原因比这更深。我见过上百个新手，他们跳过脚本通常是因为三个原因：

**原因一：不知道脚本长什么样。** 大多数教程告诉你"要写脚本"，但从来不给你看一个真实的脚本长什么样。就像告诉你要"做计划"，但从不给你看别人的计划是怎么写的。没有参照物，你当然无从下手。

**原因二：觉得脚本会限制发挥。** "我想拍到什么算什么，这样更自然。"这是新手的经典想法。但真相是：没有脚本的拍摄，拍出来的东西90%是废片。你以为的自由，其实是混乱。真正的高手是在框架内自由发挥，而不是在混乱中碰运气。

**原因三：不知道脚本该写到多细。** 有些人试过写脚本，但写着写着就卡住了——"这个镜头到底该写多细？""我需要精确到几秒吗？""台词是要一个字一个字写还是写个大概？"这些问题没有标准答案，于是干脆放弃。

让我告诉你一个事实：**所有你喜欢的视频，背后都有脚本。** 即使是那些看起来最随意的Vlog，创作者脑子里也有一个清晰的结构。区别只是：有人写在纸上，有人记在心里。而新手的问题恰恰是——你还没有能力把结构记在心里，所以你必须写下来。

写脚本不是限制你的创意，而是让你的创意有一个着力的支点。

## 二、一个完整脚本包含哪些元素

一个能直接拿去拍摄用的脚本，至少需要5个核心元素。我把它做成一个表格，你一看就懂：

| 元素 | 说明 | 示例 |
|------|------|------|
| 镜头号 | 每个镜头的编号，方便拍摄和剪辑时快速定位 | 01、02、03…… |
| 画面描述 | 这个镜头拍什么、怎么拍（景别、角度、运动方式） | 中景，从左侧45度拍摄，人物走向镜头 |
| 声音 | 这个镜头里的声音：对白、旁白、环境音、BGM | 旁白："那天我决定一个人去看海" + 海浪环境音 |
| 时长 | 这个镜头预计持续多长时间 | 3秒、5-8秒 |
| 备注 | 拍摄时需要注意的事项或备选方案 | 若光线不足可改室内拍摄，备选：用手机延时 |

这是一个最基础的脚本结构。你可以根据需要增加列，比如"拍摄地点"、"所需道具"、"情绪氛围"等，但这5列是必须有的。

有人会问："时长我怎么知道？还没拍呢。"好问题。时长不是让你精确到毫秒，而是给你一个节奏感。比如"这个镜头是快闪还是停留"，"这个画面需要让观众看几秒才能理解"。写着写着，你心里就有数了。

还有一个经常被忽略的元素：**转场方式**。两个镜头之间怎么衔接？硬切？叠化？动作接动作？如果你在脚本里提前想好，剪辑时会省掉大量纠结的时间。

## 三、三种不同类型视频的脚本模板框架

不同类型的视频，脚本的结构和重点完全不同。我给你三个模板，直接套用。

### 3.1 Vlog脚本模板

Vlog的核心是**叙事感**——用一天的时间线讲故事，而不是流水账。

```
标题：[日期] + [一句话概括今天最特别的事]

开头hook（5-10秒）
  - 镜头：当天最有画面感的一瞬间
  - 旁白/字幕：一句话引起好奇心
  - 例："你绝对想不到，今天我做了件从来没做过的事"

主体部分（按时间线或主题分3-5个段落）
  段落1：起因/出发
    - 1-2个镜头交代背景
    - 简短旁白说明今天要做什么
  段落2-4：过程/经历（重点）
    - 每段2-4个镜头
    - 交替使用：说话画面 + 空镜/特写
    - 关键：每段要有"小高潮"或"小转折"
  段落5：结果/感受
    - 回顾性旁白
    - 夕阳/夜景等氛围镜头收尾

结尾（5-10秒）
  - 一句话总结
  - 引导互动："你们觉得呢？"
```

Vlog脚本的秘诀：**不要拍你做了什么，拍你经历了什么。** "我去了一家咖啡店"是流水账，"我在一家咖啡店遇到了一个奇怪的人"才是故事。

### 3.2 宣传片脚本模板

宣传片的核心是**说服力**——每一个镜头都在回答观众的问题。

```
标题：[品牌/产品名] + 宣传片

第一幕：痛点共鸣（5-10秒）
  - 镜头：目标用户日常困扰的画面
  - 文案：说出用户的痛点
  - 例："每次整理文件，都要花半小时找那个文档"

第二幕：引出方案（5-8秒）
  - 镜头：产品/服务第一次出现
  - 文案：一句话说清这是什么
  - 注意：不要在这个阶段堆功能

第三幕：核心展示（15-30秒）
  - 2-3个使用场景，每个场景3-5个镜头
  - 每个场景解决一个具体问题
  - 画面 > 文字：让观众"看到"效果

第四幕：信任背书（5-10秒）
  - 数据/评价/用户反馈
  - 镜头：真实使用场景或数据可视化

第五幕：行动召唤（3-5秒）
  - 明确的下一步：扫码/点击/关注
  - 镜头：产品logo + 行动指引
```

宣传片的秘诀：**你的脚本不是在说"我们有多好"，而是在回答"观众为什么要care"。**

### 3.3 知识分享脚本模板

知识分享的核心是**逻辑链**——每一个知识点都是上一个的自然延伸。

```
标题：[知识点] + [价值承诺]
例："3分钟学会调色，让你的视频质感翻倍"

开头（10-15秒）
  - 问题引入：抛出一个目标受众的真实困惑
  - 价值预告：看完你能得到什么
  - 例："为什么你调出来的颜色总是灰蒙蒙的？今天3分钟帮你解决"

知识点1：核心概念（20-30秒）
  - 用比喻解释专业术语
  - 配合画面演示
  - 例："色温就像给照片加了一层滤镜，偏暖偏冷就是这个滤镜的颜色"

知识点2：操作步骤（40-60秒）
  - 分步骤，每步1-2个镜头
  - 画面：屏幕录制 + 标注
  - 每步之间用"接下来"过渡

知识点3：常见错误（15-20秒）
  - 列举1-2个新手常犯的错误
  - 错误示范 + 正确示范对比

总结 + 引导（10秒）
  - 一句话回顾核心
  - "下期我们讲xxx"
```

知识分享的秘诀：**你讲的不是知识，而是帮别人省时间。** 每个知识点都要回答"这跟我的问题有什么关系"。

## 四、"脑子里的脚本"怎么转化成可操作的分镜

很多人的脚本止步于"我想拍一个关于xxx的视频"。这一步到真正的分镜之间，隔着一个关键动作：**拆解**。

转化过程分三步：

### 第一步：把想法变成一句话

模糊的想法："我想拍一个关于咖啡的Vlog。"
一句话描述："我从没做过手冲咖啡，今天第一次尝试，结果意外地好。"

这一句话就是你的故事骨架。它包含了人物（我）、冲突（从没做过vs第一次尝试）、结果（意外地好）。

### 第二步：把一句话拆成场景

"我从没做过手冲咖啡，今天第一次尝试，结果意外地好。"

拆成场景：
- 场景1：早上，决定今天要试试手冲（起因）
- 场景2：买咖啡豆，面对一堆选择有点懵（冲突开始）
- 场景3：回家准备工具，发现家里没有滤纸（小挫折）
- 场景4：找到替代方案，开始冲泡（行动）
- 场景5：第一口品尝，表情变化（高潮）
- 场景6：对比速溶咖啡，总结感受（收尾）

### 第三步：把场景拆成镜头

以场景2"买咖啡豆，面对一堆选择有点懵"为例：

| 镜头号 | 画面描述 | 声音 | 时长 | 备注 |
|--------|----------|------|------|------|
| 05 | 远景，咖啡店门口，推门进入 | 环境音：风铃声 | 2秒 | 可用慢动作增加仪式感 |
| 06 | 中景，站在货架前，左右看 | 旁白："到了才发现，咖啡豆的分类比我写的代码还复杂" | 4秒 | 表情要迷茫 |
| 07 | 特写，手拿起一包豆子看标签 | 标签画面要清晰 | 2秒 | 可叠加字幕标注产地信息 |
| 08 | 中景，转头问店员 | 对白："第一次买，你推荐哪个？" | 3秒 | 店员反应要自然 |

关键原则：**每个镜头只做一件事。** 一个镜头里不要既有人物走动又有复杂对白，拍摄时很难执行，剪辑时也不灵活。

## 五、新手写脚本常犯的5个错误及修正方法

### 错误一：脚本写得像作文

**表现：** 脚本里全是段落式描述，没有镜头拆分。"开头我先介绍一下背景，然后展示一下过程，最后总结一下感受。"

**问题：** 这种脚本对拍摄零指导意义。你到了现场还是不知道第一个镜头该拍什么。

**修正：** 强迫自己用表格写脚本。每一行就是一个镜头。如果一句话里包含"然后"，说明你在写段落而不是在写镜头。

### 错误二：每个镜头都太长

**表现：** 脚本里一个镜头写了"拍摄整个做饭过程"，预计时长30秒。

**问题：** 30秒的单镜头在短视频时代就是催眠。观众的注意力在3-5秒就会转移。

**修正：** 把长镜头拆成多个短镜头。同一个"做饭过程"可以拆成：切菜特写2秒 + 翻锅中景3秒 + 调味特写2秒 + 成品全景2秒。4个镜头9秒，信息量远大于1个30秒的长镜头。

### 错误三：没有声音设计

**表现：** 脚本里只有画面描述，完全没有声音的规划。

**问题：** 视频是视听语言。没有声音设计，剪辑时你就只能随便找一首BGM铺上去，整个视频的节奏和情绪都是被动的。

**修正：** 每个镜头都标注：这个镜头里观众听到什么？是人物说话？环境音？还是BGM？哪里BGM起？哪里BGM落？这些在脚本里想好，剪辑效率翻倍。

### 错误四：忽略了备选方案

**表现：** 脚本写得很完美，但到了现场发现：天气不好、道具没带、场地不让拍。

**问题：** 没有备选的脚本，遇到突发情况就完全失控。你只能放弃脚本即兴拍，然后即兴拍出来的东西又回到"99%废片"的循环。

**修正：** 在备注栏写上Plan B。比如"若室外下雨则改室内窗边拍摄"，"若无法拍摄全景则改用多个局部特写拼接"。备选方案不需要详细，一句话就够，但它能让你在现场不至于手足无措。

### 错误五：脚本是死的，不敢改

**表现：** 脚本写完就当圣旨，现场发现了更好的画面也不敢拍，因为"脚本里没写"。

**问题：** 脚本是地图，不是牢笼。地图告诉你方向，但你路上看到一家有趣的店，当然可以拐进去看看。

**修正：** 建立"核心+弹性"的脚本结构。核心部分（开头hook、关键转折、结尾）严格按脚本执行。中间的过程性镜头，允许30-50%的即兴发挥。拍到计划外的好素材，加进剪辑就是了。

## 六、完整实例：从"我想记录一天"到具体分镜表

原始想法："我想记录一天。"

这是最典型的模糊起点。让我们一步步把它变成可拍摄的分镜表。

### 第一步：找到这一天里的"故事"

"记录一天"不是故事，"那天我第一次独自去了一个陌生城市"才是。

我们设定场景：周末，一个人去周边的小镇探访，第一次尝试。

故事线：出发的期待 → 路上的未知 → 到达的惊喜 → 独处的感悟 → 回程的满足

### 第二步：完整分镜表

| 镜头号 | 画面描述 | 声音 | 时长 | 备注 |
|--------|----------|------|------|------|
| 01 | 特写，闹钟显示6:30，手按掉闹钟 | 闹钟声 + 旁白："平时这个点我还在做梦，但今天不一样" | 4秒 | 画面稍暗，营造清晨感 |
| 02 | 中景，拉开窗帘，阳光涌进来 | 拉窗帘的摩擦声 + BGM渐入（轻快吉他） | 3秒 | 光线变化是关键，尽量自然光 |
| 03 | 全景，背包放在门口，检查物品 | 物品碰撞声 + 旁白："一个人出门，最怕忘带东西" | 4秒 | 背包内容特写可作为备选 |
| 04 | 中景，走出家门，回头关门 | 关门声 + BGM节奏加强 | 2秒 | 用关门声作为"出发"的节奏点 |
| 05 | 车窗外移动的风景，延时拍摄 | BGM持续 + 环境音（火车/公交） | 5秒 | 延时拍摄展现路途感 |
| 06 | 中景，到达车站，看路牌，左右张望 | 环境音（车站广播） + 旁白："到了。比我想象的小" | 4秒 | 表情要有真实的陌生感 |

| 09 | 中景，在路边摊买了一份当地小吃 | 对白："这是什么？第一次见" + 摊主回应 | 5秒 | 真实互动，不排练 |
