# 导出设置：你辛辛苦苦剪的视频，别在最后一步毁了

**剪辑花了两周，导出只花了两分钟。但这两分钟的决定，可能让之前的努力全部白费。**

---

## 前言：为什么导出设置这么重要

一个常见的场景：视频剪完，兴奋地点了导出，用了默认设置。发到平台后发现画质模糊、声音发闷、字幕被裁掉、文件大得离谱。回去改参数重新导出，又花半小时。来来回回试了三次，还是不满意。

这不是剪辑能力的问题，是导出设置没搞明白。导出是整个剪辑流程的最后一环，也是最容易被忽视的一环。参数选错了，前面所有的心血都打折扣。

这篇文章把导出设置的每个参数讲透，让你一次导出就拿到最好的结果。

---

## 一、分辨率：不是越高越好

分辨率决定了画面的清晰度，但更高的分辨率意味着更大的文件和更长的导出时间。

### 常见分辨率

| 分辨率 | 名称 | 适用场景 |
|--------|------|---------|
| 3840×2160 | 4K | 专业项目、大屏播放、需要后期裁切 |
| 2560×1440 | 2K | 中等质量需求 |
| 1920×1080 | 1080p | 大多数场景的标准选择 |
| 1280×720 | 720p | 网络直播、带宽受限场景 |
| 1080×1920 | 竖版1080p | 抖音/快手/小红书短视频 |

### 选择原则

**1. 导出分辨率 ≤ 拍摄分辨率**

如果你拍的是1080p，导出4K没有任何意义。画面不会变得更清晰，只会让文件更大。就像把一张低分辨率照片放大，不会增加细节。

**2. 匹配目标平台**

- B站/YouTube：1080p起步，4K加分
- 抖音/快手：1080p竖版（9:16）
- 微信视频号：1080p
- 朋友圈：720p够用（微信会压缩）

**3. 考虑后期空间**

如果拍摄时用了4K，但最终输出1080p，这其实是好事。4K素材导出1080p时做了下采样，画质比原生1080p更好。而且4K素材给了你在后期"数码推镜头"的空间——放大2倍画面依然清晰。

---

## 二、帧率：24fps还是30fps还是60fps

帧率决定了画面的流畅度。选错帧率，要么画面卡顿，要么文件浪费。

### 常见帧率

| 帧率 | 视觉感受 | 适用场景 |
|------|---------|---------|
| 24fps | 电影感、微顿 | 剧情片、纪录片、氛围视频 |
| 25fps | 电视标准（PAL制式） | 国内电视播出 |
| 30fps | 流畅、日常 | Vlog、教程、大部分网络视频 |
| 60fps | 非常流畅 | 运动、游戏、慢动作素材 |

### 选择原则

**1. 导出帧率 = 拍摄帧率**

拍的时候用的30fps，导出就选30fps。不要把30fps的素材导出为60fps，软件会通过插帧"凭空制造"画面，可能导致画面伪影。

**2. 24fps的电影感不是万能的**

24fps有轻微的顿挫感，这在电影里是"质感"，但在教程或Vlog里可能让人觉得"卡"。大部分网络内容用30fps更安全。

**3. 慢动作需要高帧率**

如果你拍了一段60fps的素材，想做成2倍慢动作，导出时帧率设为30fps。这样60帧的素材播30帧每秒，自然就是2倍慢放，画质不损失。如果用30fps素材做慢动作，软件只能重复帧，画面会卡。

**4. 不同帧率素材混剪**

如果项目中混合了不同帧率的素材（比如主体30fps，慢动作片段60fps），导出时以主体素材帧率为准。大多数剪辑软件会自动处理帧率转换。

---

## 三、码率：画质与文件大小的平衡术

码率是导出设置里最关键也最容易被忽略的参数。它直接决定了画质的清晰度和文件的大小。

### 什么是码率

码率（Bitrate）是每秒视频数据的量，单位是Mbps（兆比特每秒）。码率越高，画质越好，文件越大。码率越低，文件越小，画质越差。

**一个直观对比：**

- 1080p 5Mbps → 文件小，画面有明显的色块和模糊
- 1080p 10Mbps → 画质不错，大部分场景够用
- 1080p 20Mbps → 画质很好，文件较大
- 1080p 50Mbps → 画质极好，文件很大

### CBR vs VBR

**CBR（固定码率）**：整个视频码率不变。简单场景浪费空间，复杂场景可能不够用。适合直播推流。

**VBR（可变码率）**：根据画面复杂度自动调整码率。静态画面用低码率，运动画面用高码率。大部分场景推荐VBR。

**VBR的两种模式：**
- **1-pass**：分析一遍后编码，速度快
- **2-pass**：分析两遍后编码，画质更好，速度慢

推荐选择：**VBR 2-pass**（如果你不赶时间），**VBR 1-pass**（如果赶时间）。

### 推荐码率设置

| 分辨率 | 推荐码率（VBR） | 最低码率 |
|--------|----------------|---------|
| 720p | 5-8 Mbps | 3 Mbps |
| 1080p | 10-15 Mbps | 5 Mbps |
| 2K | 15-25 Mbps | 8 Mbps |
| 4K | 30-60 Mbps | 15 Mbps |

如果你的视频有很多运动画面（体育、舞蹈、快速切换），码率取高值。如果是静态画面为主（教学、访谈），取低值即可。

## 四、编码格式：H.264、H.265还是ProRes

编码格式决定了视频的压缩方式和兼容性。选错了，要么文件巨大，要么平台不认。

### 三种主流编码

**H.264（AVC）**
- 兼容性最好，几乎所有设备和平台都支持
- 压缩效率中等，文件大小适中
- 编码速度快
- **推荐：网络发布的默认选择**

**H.265（HEVC）**
- 压缩效率比H.264高约40%，同样画质文件更小
- 兼容性不如H.264，部分老设备不支持
- 编码速度较慢
- **推荐：需要小文件高画质的场景**

**ProRes**
- Apple开发的专业编码，几乎无损
- 文件非常大（1080p一分钟约1GB）
- 剪辑时流畅，适合中间编辑环节
- 不适合最终发布
- **推荐：后期制作过程中的中间格式**

### 选择建议

| 场景 | 编码 | 理由 |
|------|------|------|
| 发B站/抖音/微信 | H.264 | 兼容性最好 |
| 追求画质+文件小 | H.265 | 同画质体积小40% |
| 交付给其他剪辑师 | ProRes 422 | 无损传递 |
| 自用存档 | H.265 | 节省空间 |
| 不确定用什么 | H.264 | 最安全 |

## 五、音频设置：声音也值得认真对待

视频的音频经常被忽视，但糟糕的音频体验会毁掉整个视频。

### 推荐音频设置

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 采样率 | 48kHz | 视频标准采样率 |
| 声道 | 立体声 | 大部分场景 |
| 音频编码 | AAC | 兼容性最好 |
| 音频码率 | 320kbps | 高品质 |
| 最低码率 | 128kbps | 语音内容可用 |

### 音量标准化

导出前确保音量在合理范围内。人声峰值建议在-6dB到-3dB之间，整体音量不超过0dB（会爆音）。大部分剪辑软件都有音频电平表，红色区域就是爆音。

**实操：** 在导出前播放整个视频，观察音频电平表。如果有红色峰值，降低对应片段的音量。

---

## 六、容器格式：MP4还是MOV

容器格式（文件后缀）决定了视频文件的封装方式。

| 格式 | 特点 | 适用场景 |
|------|------|---------|
| MP4 | 兼容性最好，H.264/H.265+AAC | 网络发布、通用 |
| MOV | Apple生态，ProRes+AAC | Mac用户、后期中间格式 |
| WebM | 开源，VP9/AV1 | 网页嵌入 |
| MKV | 支持多音轨多字幕 | 存档 |

**推荐：** 99%的情况选MP4。它是网络发布的通用格式，所有平台都支持。

## 七、三款软件导出设置速查

### 剪映导出推荐

```
分辨率：1080p（1920×1080）或 4K
帧率：30fps
编码：H.264（默认）
码率：选"推荐"或"更高"
音频：AAC 48kHz
格式：MP4
```

剪映的导出界面比较简洁，大部分参数用默认就行。关键是选对分辨率和帧率。

### Premiere Pro导出推荐

```
格式：H.264
预设：匹配源 - 高比特率（Match Source - High Bitrate）
分辨率：与序列设置一致
帧率：与序列设置一致
码率：VBR 2-pass，目标10Mbps（1080p）
音频：AAC，48kHz，320kbps，立体声
```

PR的导出设置项很多，但掌握"H.264 + 匹配源 + VBR 2-pass"这个组合就够了。

### Final Cut Pro导出推荐

```
格式：H.264（发布用）或 ProRes 422（中间格式）
分辨率：与项目设置一致
码率：选"更好质量"
音频：AAC，48kHz，320kbps
```

FCP的导出叫"共享"（Share），设置相对简单。重点是区分"发布用"和"编辑用"。

## 八、平台专项适配

不同平台有不同的推荐参数和压缩策略。了解平台的特性，可以在导出时做出更好的选择。

### B站

- 推荐分辨率：1080p或4K
- 推荐码率：1080p不低于8Mbps，4K不低于20Mbps
- B站会对上传视频做二次压缩，所以导出码率可以比平台推荐值高20-30%，留出压缩余量
- 竖版视频（9:16）在B站播放器里会有大量黑边，不推荐

### 抖音

- 推荐：1080×1920（9:16竖版）
- 码率：8-12Mbps
- 时长：15秒-3分钟最佳
- 抖音压缩较激进，导出码率建议取高值

### YouTube

- 推荐：1080p或4K，16:9横版
- 码率：1080p 10-15Mbps，4K 35-60Mbps
- YouTube的压缩算法较优，原始画质越高，压缩后效果越好

### 微信视频号

- 推荐：1080p
- 微信压缩非常狠，导出时码率取高值（15-20Mbps）
- 文件大小限制1GB

## 九、导出前终极检查清单

每次点"导出"之前，过一遍这个清单：

- [ ] 分辨率匹配拍摄和目标平台
- [ ] 帧率与拍摄一致
- [ ] 码率设置合理（参考上方推荐值）
- [ ] 编码格式选对（H.264用于发布）
- [ ] 音频设置正确（48kHz/AAC/320kbps）
- [ ] 音量无爆音（峰值不超过0dB）
- [ ] 容器格式选MP4（通用发布）
- [ ] 画面无黑边、无裁切
- [ ] 字幕安全区检查（字幕不会被平台UI遮挡）
- [ ] 导出后播放检查一遍，确认无问题

**导出后必做：** 打开导出的文件，从头到尾看一遍。不要只看开头就发出去。有时候导出过程中会出现音画不同步、画面闪烁、字幕缺失等问题，只有完整播放才能发现。
