# 小红书评论爬虫优化方案

## 当前问题分析

### 1. 爬虫功能不完整
- ❌ 没有获取子评论（回复评论）的功能
- ❌ 游标分页处理不够完善
- ❌ 防爬措施简单（只有固定3秒延迟）

### 2. 防爬措施不足
- ❌ 缺少随机延迟
- ❌ 缺少重试机制
- ❌ 缺少代理支持
- ❌ 缺少请求头随机化

### 3. LLM 集成缺失
- ❌ 当前使用模拟数据
- ❌ 没有真实的大模型调用

---

## 优化方案

### 一、爬虫模块优化

#### 1.1 支持获取子评论（互相评论）

小红书 API 说明：
- 主评论接口：`/api/sns/web/v1/comment/page`
- 子评论接口：`/api/sns/web/v1/comment/sub/page`

#### 1.2 增强防爬措施

| 措施 | 说明 |
|------|------|
| **随机延迟** | 请求间隔在 2-5 秒之间随机 |
| **指数退避重试** | 失败时等待时间递增：1s, 2s, 4s, 8s |
| **User-Agent 池** | 随机切换不同的浏览器 UA |
| **代理支持** | 支持配置代理池 |
| **请求指纹保护** | 避免请求模式过于规律 |

#### 1.3 数据结构优化

```python
# 评论数据结构
{
    "id": "评论ID",
    "content": "评论内容",
    "user_name": "用户名",
    "user_id": "用户ID",
    "create_time": "创建时间",
    "liked_count": 点赞数,
    "reply_count": 回复数,
    "sub_comments": [  # 子评论列表
        {
            "id": "子评论ID",
            "content": "子评论内容",
            "user_name": "用户名",
            "target_user_name": "回复的用户名",
            "create_time": "创建时间",
            "liked_count": 点赞数
        }
    ]
}
```

## 使用指南

### 1. 获取 Cookie

1. 浏览器登录小红书
2. F12 打开开发者工具
3. Application → Cookies → xiaohongshu.com
4. 复制以下关键 Cookie：
   - `a1`
   - `webId`
   - `web_session` (如果有)

### 2. 配置使用

```python
from app.scraper_optimized import XiaohongshuScraperOptimized

# 初始化爬虫
scraper = XiaohongshuScraperOptimized(
    cookies="你的Cookie字符串",
    use_random_delay=True,  # 启用随机延迟
    max_retries=3,           # 最大重试次数
    proxy=None                # 代理地址（可选）
)

# 获取所有评论（包含子评论）
all_comments = scraper.get_all_comments_with_sub(
    note_id="笔记ID",
    max_pages=10,
    include_sub_comments=True
)

print(f"共获取 {len(all_comments)} 条主评论")
```

## 防爬最佳实践

### 1. 请求间隔

| 场景 | 推荐间隔 |
|------|---------|
| 单页请求 | 2-5 秒随机 |
| 子评论请求 | 1-3 秒随机 |
| 大规模抓取 | 5-10 秒随机 |

### 2. Cookie 维护

- Cookie 有效期约 1-2 周
- 建议使用备用账号轮换
- 检测到验证码时暂停抓取

### 3. 代理使用

如果需要大规模抓取，建议：
- 使用高质量代理池
- 每个 IP 每天限制请求次数
- 代理轮换间隔 ≥ 30 秒

## LLM 集成方案

### 1. 提示词设计

```python
SYSTEM_PROMPT = """
你是一个专业的评论分析助手。请从用户提供的小红书评论中提取以下信息：

1. 【有用总结】：提取真实用户体验、产品优缺点、使用感受
2. 【避坑指南】：识别负面评价、质量问题、服务问题、警告信息
3. 【待确认】：信息不明确、需要进一步验证的问题

评论数据中包含主评论和子评论（用户之间的互相回复），请综合分析。

输出格式：JSON，结构如下：
{
    "summary": ["总结1", "总结2"],
    "pitfalls": ["避坑1", "避坑2"],
    "uncertain": ["疑问1", "疑问2"]
}

注意：
- 优先提取点赞数高的评论
- 子评论中的补充信息也要考虑
- 保持客观，不要添加主观推测
"""
```

### 2. 分批处理策略

如果评论数量很多（> 100条）：
1. 按点赞数排序，取前 50-100 条
2. 分批输入 LLM，每批 20-30 条
3. 最后合并结果
