## 1. Problem Statement（问题陈述）

用户在社交平台（小红书、大众点评、抖音等）看到一篇热门内容时，评论区往往有几百甚至上千条评论。用户想要从中提取有价值的信息（如：真实体验、避坑建议、产品优缺点），但需要花费大量时间逐条翻阅。

**用户痛点**:
- 翻评论太累，尤其热门内容评论区
- 有用的信息分散，难以快速提取
- 负面评价（避坑）和真实体验被淹没

**不解决的代价**:
- 用户决策效率低，可能错过关键信息
- 对产品/店铺产生错误预期

## 4. User Personas & Stories（用户画像与故事）

### 主要用户画像

**画像 1**: 年轻消费者（22-35岁）
- 决策前会参考其他用户评价
- 追求效率，不愿花大量时间翻评论
- 使用场景：餐厅选择、旅行目的地、产品购买

**画像 2**: 内容创作者/博主
- 需要快速了解某话题的群众声音
- 收集素材、选题参考

---

### 核心用户故事

**Story 1**: 作为消费者，我想要粘贴一个分享链接，然后自动看到评论区的精华总结，以便快速做出消费决策。

**验收标准**:
- [ ] Given 用户粘贴小红书链接, when 点击分析, then 显示"有用总结""避坑指南""待确认"三个分类
- [ ] Given 评论数 > 500, when 分析, then 10 秒内完成
- [ ] Given 用户点击"避坑指南", then 显示所有负面/警告类评论

**Story 2**: 作为创作者，我想要看到某个话题下评论的总体舆情，以便了解大众对某产品/话题的真实看法。

**验收标准**:
- [ ] Given 用户输入话题关键词, when 搜索, then 显示相关热门内容列表
- [ ] Given 选择某条内容, then 显示评论情感分布（正面/中性/负面）

## 5. Solution Overview（方案概述）

### 产品形态
- **V1**: Web 应用（PC端为主）
- **后期**: 微信小程序（用户明确方向）

### 核心功能

| 功能 | 描述 |
|------|------|
| **链接解析** | 输入小红书/大众点评/抖音分享链接，自动提取内容ID |
| **评论抓取** | 调用平台API获取评论区数据 |
| **AI 提炼** | 使用 LLM 对评论进行分类、摘要、情感分析 |
| **结果展示** | 分类呈现：有用的总结、避坑指南、存疑项 |

### 技术架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   前端      │ ──▶ │   后端 API  │ ──▶ │  评论数据源 │
│  (React)    │     │  (FastAPI)  │     │  (平台API)  │
└─────────────┘     └─────────────┘     └─────────────┘
                           │
                           ▼
                    ┌─────────────┐
                    │   LLM 提炼   │
                    │ (GPT/Claude) │
                    └─────────────┘
```

### 第一阶段技术栈

- **前端**: React + Vite（简单页面）
- **后端**: Python FastAPI
- **数据库**: SQLite（本地）或 PostgreSQL
- **部署**: 本地运行 / 简单 VPS
