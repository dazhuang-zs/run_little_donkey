## 1. 项目概述

| 项目 | 说明 |
|------|------|
| **名称** | 评论洞察助手 (Comment Insight) |
| **目标** | 解决用户在社交平台评论区提取有价值信息效率低的问题 |
| **输入** | 小红书/大众点评/抖音分享链接 |
| **输出** | 有用总结、避坑指南、存疑标记 |

## 3. 核心模块分析

### 3.1 链接解析模块

**功能**：识别输入的分享链接类型，提取内容 ID

**支持的平台**：

| 平台 | 链接示例 | 提取内容 |
|------|----------|----------|
| 小红书 | `https://www.xiaohongshu.com/explore/xxx` | 笔记 ID |
| 大众点评 | `https://www.dianping.com/shop/xxx` | 店铺 ID |
| 抖音 | `https://v.douyin.com/xxx` | 视频 ID |

**实现方案**：
- 正则表达式匹配
- 各平台短链解析
- 统一输出：`(platform, content_id)` 元组

### 3.2 评论抓取模块

**挑战**：各平台反爬机制

| 平台 | 难度 | 可行方案 |
|------|------|----------|
| 小红书 | 中 | 模拟请求 + 代理池 |
| 大众点评 | 高 | 账号模拟登录 |
| 抖音 | 高 | 开放 API / 第三方 SDK |

**MVP 策略**：
- 先实现小红书（需求最明确）
- 使用模拟数据 Demo 验证产品
- 后期接入真实 API

### 3.3 LLM 处理模块

**工作流程**：

```
评论数据 → 预处理 → 分批 → LLM 提炼 → 结果聚合 → 分类输出
```

**Prompt 设计**：

```python
SYSTEM_PROMPT = """
你是一个评论分析助手。根据用户提供的评论数据，请进行以下分析：
1. 有用总结：提取真实体验、产品优缺点
2. 避坑指南：识别负面评价、警告信息
3. 待确认：信息不明确、需要进一步验证的内容

输出格式：JSON 数组
"""
```

### 3.4 数据存储

**数据库设计**：

| 表名 | 用途 |
|------|------|
| `content` | 原始内容存储 |
| `comments` | 评论原始数据 |
| `analysis_results` | 分析结果缓存 |
| `users` | 用户信息（可选） |
