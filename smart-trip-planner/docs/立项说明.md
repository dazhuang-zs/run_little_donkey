## 2. 项目目标

**一句话描述**：用户用自然语言描述旅行需求，AI自动解析意图，调用腾讯地图API完成POI检索、路径规划、可视化展示，输出完整行程方案。

**参赛角度**：
- 活动明确提到QClaw为推荐AI工具（官方加分项）
- 展示AI Agent × MCP × 腾讯地图的完整技术链
- 有可运行Demo（评审加分项）

## 3. 核心功能

### 3.1 AI意图解析
- 输入：自然语言（如"北京3日游，第一天故宫+天安门，住王府井附近"）
- 输出：结构化JSON（城市、天数、景点列表、住宿偏好、预算等）
- 实现：DeepSeek-V3/硅基流动等大模型API + Prompt工程

### 3.2 POI智能检索
- 腾讯地图WebService API：POI搜索、地理编码、逆地理编码
- 同名消歧：如"故宫"→ 北京故宫博物院
- 周边搜索：推荐住宿、餐厅

### 3.3 多点路径优化
- 距离矩阵API计算所有景点间距离
- 贪心/遗传算法优化游览顺序
- 支持驾车/公交/步行多种出行方式

### 3.4 可视化展示
- 腾讯地图JS API（GL版）展示地图
- 路线标注、景点标记、住宿区域圈定
- 交互式：点击景点查看详情

### 3.5 行程报告生成
- 结构化行程表（时间、地点、交通方式、预估耗时）
- 推荐列表（酒店/餐厅/周边景点）
- 支持导出/分享

## 6. QClaw在项目中的角色

QClaw（OpenClaw）是本项目的**开发助手**，而非运行时AI服务。

**QClaw 负责的开发工作：**
- 代码编写与重构（本项目后端即由 QClaw 协助开发）
- 调试与问题排查
- 文档生成与优化
- 架构设计与评审

**运行时的AI能力层：**
- DeepSeek-V3（性价比最高，$0.27/M tokens）
- 硅基流动（国内直达，支持 DeepSeek/Qwen）
- OpenAI GPT 系列（可选）

### 6.1 MCP工具链（运行时架构）
将腾讯地图API封装为MCP工具，Agent通过Tool Calling调用：

```
LLM Agent（DeepSeek/硅基流动）
  ├─ mcp__tencentmap__search_poi(keyword, city)
  ├─ mcp__tencentmap__geocode(address)
  ├─ mcp__tencentmap__distance(from, to, mode)
  ├─ mcp__tencentmap__direction(from, to, mode)
  └─ mcp__tencentmap__reverse_geocode(lat, lng)
```

### 6.2 QClaw开发 + LLM运行的双层架构

```
开发阶段：QClaw（你与我）→ 编写代码、设计架构、调试问题
运行阶段：DeepSeek/硅基流动 → 意图解析、对话Agent、报告生成
```
