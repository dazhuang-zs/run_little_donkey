# 智能行程规划助手 — 产品设计文档

> 本文档为智能行程规划助手（Smart Trip Planner）的产品设计稿，从产品经理视角阐述产品定位、功能规划、用户旅程及成功指标。
> 活动关联：腾讯位置服务开发者征文大赛（CSDN id=11152）

## 一、产品愿景

### 1.1 一句话定位

**用自然语言描述旅行需求，AI 自动生成完整行程方案。**

### 1.2 解决的核心问题

| 痛点 | 现状 | 我们的方案 |
|------|------|------------|
| 规划旅行费时 | 多个 App 切换（地图/酒店/景点/餐厅）| 一站式完成，AI 统筹 |
| 路线安排靠猜 | 不确定景点间距离，不知道先后顺序 | 腾讯地图距离 API + 贪心算法优化 |
| 不知道住哪方便 | 靠攻略推荐，不一定适合自己的行程 | 基于景点中心点推荐住宿区域 |
| 出行方式不清晰 | 公交/驾车/步行各有优劣 | API 获取多模式路线，用户自选 |

### 1.3 产品价值主张

```
输入一句话 → 得到完整行程
   ↓
"北京3日游，第一天故宫+天安门，
  第二天长城，住王府井附近，预算500元/晚"
   ↓
[城市]北京 [天数]3天 [景点]故宫/天安门/长城
[住宿]王府井附近 [预算]500元/晚
   ↓
自动规划路线 → 可视化地图 → 行程报告
```

## 二、目标用户

### 2.1 主要用户画像

| 用户类型 | 特征 | 核心诉求 |
|---------|------|---------|
| 自由行游客 | 喜欢自主安排行程，厌恶跟团 | 快速规划，不用研究地图 |
| 出差顺便旅游 | 时间有限，想高效利用 | 最优路线推荐，节省时间 |
| 家庭出游 | 多景点、多成员，需求复杂 | 路线规划 + 住宿便利性 |
| 学生党 | 预算有限，偏好穷游 | 低成本路线 + 公共交通优先 |

### 2.2 用户真实场景

**场景 1：小明周末去北京**
> 小明计划周末去北京玩两天一夜。他打开产品，输入："北京2日游，第一天故宫+天安门，第二天长城，住前门附近，预算300元/晚"。AI 自动生成了：
> - 每天的景点游览顺序
> - 各景点间的交通方式
> - 前门附近推荐的平价酒店
> - 每段路程的预估时间
> - 完整的可视化地图

**场景 2：产品经理出差**
> 产品经理王姐要去深圳出差 3 天，结束后有半天自由时间。她输入："深圳3天，前两天开会，第三天下午自由活动，主要想去世界之窗和东门老街，预算400元/晚"。AI 帮她规划了前两天紧凑的商务路线 + 第三天轻松的游览路线。

## 三、核心功能规格

### 3.1 功能优先级矩阵

| 功能 | 用户痛点 | 技术实现 | 优先级 | 备注 |
|------|---------|---------|--------|------|
| 自然语言输入 | 手动输入太麻烦 | AI意图解析 | P0 | 核心差异化 |
| POI搜索 | 不知道搜什么 | 腾讯地图API | P0 | 基础能力 |
| 路线规划 | 手动排顺序 | 距离矩阵+贪心算法 | P0 | 技术亮点 |
| 地图可视化 | 文字不直观 | 腾讯地图JS API | P0 | 视觉呈现 |
| 住宿推荐 | 不知道住哪 | 周边搜索+AI推荐 | P1 | 增强体验 |
| 行程报告 | 需要纸质记录 | 报告生成器 | P1 | 交付物 |
| 交通模式切换 | 出行方式不确定 | 多模式API | P2 | 锦上添花 |
| 分享功能 | 分享给同行人 | URL/图片导出 | P2 | 社交传播 |

> **P0**：MVP 必须完成，影响核心体验  
> **P1**：正式版应具备，提升竞争力  
> **P2**：后续迭代添加

### 3.2 P0 功能详细规格

#### F1：自然语言输入

**输入示例：**

```
"北京3日游，第一天故宫+天安门，第二天长城，
住王府井附近，预算500元/晚，想坐地铁"

"上海2天，想去外滩+城隍庙+陆家嘴，吃本帮菜"

"广州3日，第一天长隆，第二天沙面，第三天早茶+粤语，
预算600元/晚"
```

**AI 解析目标：**
- `city`：城市名
- `days`：天数
- `attractions[]`：景点列表
- `hotel_area`：住宿区域偏好
- `budget`：预算（元/晚）
- `transport_mode`：交通偏好（驾车/公交/步行）

**解析策略：**
1. 规则匹配 + 正则表达式（快速解析城市、天数）
2. AI 大模型补充解析景点、偏好（意图理解）
3. 用户确认纠错（表单回显）

#### F2：POI 智能检索

**流程：**

```
用户景点 → 城市限定搜索 → 腾讯地图 POI API
    → 结果排序（评分/距离/热度）
    → 同名消歧（城市+类别双重限定）
    → 候选 POI 列表
```

**同名消歧策略：**
- "故宫" → 优先返回北京故宫博物院（城市限定）
- "王府井" → 返回商业区/地铁站坐标（地标优先）
- 多个同名 → 按用户所在区域相关性排序

**周边推荐：**
- 住宿区域中心点计算 → 搜索周边酒店
- 景点密集区 → 搜索周边餐厅

#### F3：多点路线优化

**算法选择：**

| 算法 | 时间复杂度 | 适用规模 | 实现难度 |
|------|-----------|---------|---------|
| 贪心最近邻 | O(n²) | ≤10个景点 | ⭐ 简单 |
| 2-opt 局部优化 | O(n²×迭代次数) | ≤20个景点 | ⭐⭐ 中等 |
| 遗传算法 | O(种群×迭代×n²) | 任意规模 | ⭐⭐⭐ 复杂 |

**MVP 采用贪心最近邻算法：**
```
输入：景点列表 P = [P1, P2, ..., Pn]
输出：游览顺序 Route

1. 计算所有景点间的距离矩阵 D[n][n]
2. 选择起点（第一个景点）
3. 循环：选距离当前点最近的未访问景点
4. 直到所有景点访问完毕
```

**技术细节：**
- 腾讯地图 Distance API：批量计算距离矩阵
- 缓存层：相同城市景点间距离缓存 24 小时
- 限流：距离矩阵计算量较大，加入请求队列

#### F4：地图可视化

**前端展示内容：**

```
地图层
  ├─ 景点标记（红色 Pin，带名称标注）
  ├─ 路线连线（蓝色虚线，按游览顺序）
  ├─ 住宿标记（绿色 Pin，带价格标注）
  ├─ 餐厅标记（橙色 Pin）
  └─ 交通线（根据模式：实线=驾车/虚线=公交/点线=步行）
```

**交互功能：**
- 点击景点标记 → 显示详情卡片（名称、开放时间、门票、简介）
- 点击路线 → 显示该段交通详情（距离、耗时、交通方式）
- 切换交通模式 → 路线重新渲染

| 节点 | 预期行为 | 异常处理 |

## 七、LLM + MCP 集成方案

### 7.1 架构定位

本产品运行时使用 **DeepSeek-V3 / 硅基流动** 作为 AI 能力层，通过 MCP 协议调用腾讯地图 API：

```
用户输入 → LLM（DeepSeek）解析意图 → 结构化行程要素
    ↓
LLM Agent → MCP 工具链 → 腾讯地图 API
    ↓
返回结果 → LLM 生成行程报告 → 用户
```

**QClaw 在本项目中的角色**：开发助手（负责代码开发、架构设计、文档撰写），而非运行时 AI 服务。

### 7.2 MCP 工具定义

| 工具名 | 功能 | 参数 | 返回 |
|--------|------|------|------|
| `tencentmap_poi_search` | POI 搜索 | keyword, city, category | POI 列表 |
| `tencentmap_geocode` | 地址→坐标 | address | lat, lng |
| `tencentmap_distance` | 距离计算 | from_lat, from_lng, to_lat, to_lng, mode | 距离, 耗时 |
| `tencentmap_direction` | 路线规划 | from, to, mode | 路线坐标点 |
| `tencentmap_reverse_geocode` | 坐标→地址 | lat, lng | 地址描述 |

### 7.3 LLM Prompt 设计

```
你是智能行程规划助手。用户会输入旅行需求，
你通过调用 MCP 工具链获取腾讯地图数据，
最终输出结构化的行程方案。

工作流程：
1. 解析用户输入，提取城市、天数、景点
2. 调用 poi_search 搜索各景点坐标
3. 调用 distance 计算景点间距离
4. 调用 direction 获取路线
5. 综合以上信息，生成行程方案

输出格式：
{
  "city": "城市名",
  "days": 天数,
  "itinerary": [
    {
      "day": 1,
      "attractions": [
        {"name": "景点名", "lat": 39.9, "lng": 116.4, "arrival": "09:00", "departure": "11:00"}
      ],
      "transport_to_next": {"mode": "地铁", "duration": 30}
    }
  ],
  "hotel": {"name": "酒店名", "lat": ..., "lng": ..., "price": 300}
}
```
