## 三、PK 系统与人物属性优化

### 3.1 当前问题诊断

| 问题 | 影响 | 优先级 |
|------|------|--------|
| 六维属性不够直观 | 用户难理解战力差距 | 高 |
| PK过程缺乏悬念 | 观看体验平淡 | 高 |
| 没有技能/羁绊系统 | 缺少策略深度 | 中 |
| 缺少养成元素 | 用户粘性不足 | 中 |

### 3.2 核心优化方案

#### A. 战力可视化升级

**当前**：纯数字六维
**优化**：综合战力指数 + 特色标签

```javascript
// 人物卡片新设计
{
  name: "诸葛亮",
  title: "卧龙先生",
  power: 96,           // 综合战力（自动计算）
  tags: ["智谋型", "统帅型", "军师"],  // 特色标签
  skill: "空城计",      // 主动技能
  passive: "鞠躬尽瘁",  // 被动技能
  synergy: ["刘备", "关羽", "张飞"]  // 羁绊人物
}
```

**战力计算公式**：
```
综合战力 = 统帅×0.25 + 武力×0.2 + 智谋×0.25 + 政治×0.15 + 魅力×0.1 + 技艺×0.05

特色标签规则：
- 武力型：武力 > 90
- 智谋型：智谋 > 90
- 统帅型：统帅 > 90
- 全能型：无短板（每项 > 70）
- 偏科型：某一项 > 95 且有短板 < 50
```

#### B. 技能系统（增加策略性）

**技能类型**：
| 类型 | 效果 | 示例 |
|------|------|------|
| 主动技能 | 回合触发，改变战局 | 空城计：低血量时免疫一次致命伤害 |
| 被动技能 | 常驻效果 | 精忠报国：血量低于30%时攻击+20% |
| 羁绊技能 | 特定组合激活 | 桃园结义：刘关张同队，全属性+10% |
| 省份Buff | 地域特色 | 中原腹地：河南队伍开局士气+10 |

**技能展示方式**：
- 视频：技能触发时全屏特效 + 文字说明
- 游戏：鼠标悬停显示技能详情

#### C. PK 过程戏剧化

**当前流程**：
```
双方出场 → 数值对比 → 胜负揭晓
```

**优化流程**：
```
1. 开场介绍（3秒）
   - 双方省份旗帜动画
   - 出战人物走马灯

2. 战力对比（5秒）
   - 六维雷达图对决
   - 综合战力数字跳动

3. 回合制PK（核心）
   - 每回合随机抽取属性对决
   - 高战力有优势，但低战力可暴击
   - 技能触发关键时刻

4. 高潮时刻（3秒）
   - 血量低于20%触发"背水一战"
   - 随机事件（天气、地形影响）

5. 胜负揭晓（5秒）
   - 胜利方省份高亮
   - 失败方颜色渐变到胜利方
   - 战力吸收动画
```

**随机事件系统**：
```javascript
const randomEvents = [
  { name: "天降大雨", effect: "武力型-10%", probability: 0.1 },
  { name: "粮草充足", effect: "统帅型+10%", probability: 0.1 },
  { name: "士气高涨", effect: "随机一方首回合暴击", probability: 0.1 },
  { name: "内讧", effect: "人数多的一方-5%", probability: 0.05 }
];
```
