## Day 1：API调用与Prompt工程

### 目标

不读任何理论，先让大模型给你"干活"。

### 任务

**任务1：获取API Key**

注册以下平台（选2个就够了）：
- DeepSeek（国内最便宜，性价比高）
- 阿里云百炼（通义千问，有免费额度）
- OpenRouter（聚合多个模型，有免费额度）

拿到API Key后，存到环境变量里：
```bash
export DEEPSEEK_API_KEY="sk-xxx"
```

**任务2：第一个API调用**

用Python写一个最简单的调用：
```python
import requests

url = "https://api.deepseek.com/v1/chat/completions"
headers = {
    "Authorization": f"Bearer {API_KEY}",
    "Content-Type": "application/json"
}
data = {
    "model": "deepseek-chat",
    "messages": [
        {"role": "user", "content": "用一句话解释什么是RAG"}
    ]
}

response = requests.post(url, headers=headers, json=data)
print(response.json()["choices"][0]["message"]["content"])
```

跑通这个，你就已经比90%的人走得更远了。

**任务3：写3个有效Prompt**

尝试这三个场景：
1. **总结**：给一篇长文章，让模型总结成3个要点
2. **改写**：给一段文字，让模型改成"小红书风格"
3. **推理**：给一个简单逻辑题，看模型能不能推理出来

记录每个Prompt的效果，思考：什么样的指令效果更好？

### 今天的认知更新

- Prompt不是"随便问问"，而是**精确的指令设计**
- 好的Prompt包含：角色、任务、约束、示例

## Day 2：模型能力边界与选择

### 目标

知道大模型能做什么，不能做什么，以及不同模型的差异。

### 任务

**任务1：测试模型边界**

准备5个测试题：
1. 数学计算：`12345 × 67890 = ?`
2. 逻辑推理：经典的"小明比小红大2岁，小红比小刚大3岁..."
3. 知识问答：一个具体领域的专业问题
4. 代码生成：写一个Python函数解决某问题
5. 长文本理解：给一段5000字的文章，问细节

用同一个Prompt，分别测试3个模型（如DeepSeek、通义、GPT），记录：
- 哪个回答正确？
- 哪个更详细？
- 哪个更快？

**任务2：理解模型的"幻觉"**

问模型一个你确定答案的问题（比如你领域的专业知识），看它会编造吗？

认知更新：**模型不知道自己不知道什么**。它会自信地给出错误答案。

### 今天的认知更新

- 不同模型有不同擅长的领域
- 数学、逻辑推理是普遍弱项
- 模型会"幻觉"，答案需要验证

## Day 3：RAG基础——让模型"读"你的数据

### 目标

大模型的知识有截止日期，且不知道你私有的数据。RAG（检索增强生成）就是解决这个问题的。

### 任务

**任务1：理解RAG原理**

简单说，RAG就是：
1. 把你的文档切成小块（Chunk）
2. 用向量模型把每块转成向量（Embedding）
3. 用户提问时，检索最相关的几块
4. 把这几块作为上下文喂给模型

**任务2：搭建第一个RAG**

用LangChain（Python最流行的框架）：
```python
from langchain.vectorstores import Chroma
from langchain.embeddings import HuggingFaceEmbeddings
from langchain.document_loaders import TextLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter

# 1. 加载文档
loader = TextLoader("my_document.txt")
docs = loader.load()

# 2. 切分
text_splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
chunks = text_splitter.split_documents(docs)

# 3. 向量化
embeddings = HuggingFaceEmbeddings()
vectorstore = Chroma.from_documents(chunks, embeddings)

# 4. 检索
results = vectorstore.similarity_search("你的问题", k=3)
for doc in results:
    print(doc.page_content)
```

**任务3：验证效果**

准备一篇你领域的技术文档，问几个细节问题。对比：
- 直接问模型（没有RAG）
- 用RAG

看答案差异。

### 今天的认知更新

- RAG不是魔法，是"检索+生成"的组合
- 切分策略、向量模型质量，决定RAG效果

## Day 4：Agent入门——让模型"做"事

### 目标

大模型不只是"问答机"，它可以调用工具、执行多步任务——这就是Agent。

### 任务

**任务1：理解Agent原理**

Agent的核心循环：
1. 思考：我应该做什么？
2. 行动：调用工具
3. 观察：看返回结果
4. 重复：继续思考...

这就是ReAct框架（Reasoning + Acting）。

**任务2：实现一个简单Agent**

用LangChain的Agent：
```python
from langchain.agents import initialize_agent, Tool
from langchain.llms import OpenAI

# 定义工具
tools = [
    Tool(
        name="Calculator",
        func=lambda x: str(eval(x)),
        description="用于数学计算"
    )
]

# 创建Agent
llm = OpenAI(temperature=0)
agent = initialize_agent(tools, llm, agent="zero-shot-react-description")

# 运行
agent.run("计算 (23 + 17) × 5")
```

观察：Agent会自己判断要不要调用计算器工具。

### 今天的认知更新

- Agent不是模型能力，是"模型+工具+循环"的设计模式
- 工具定义、Prompt设计，决定Agent效果

## Day 5：微调基础——让模型更"懂"你

### 目标

微调（Fine-tuning）是在基础模型上，用特定数据继续训练，让模型更适合你的场景。

### 任务

**任务1：理解微调的类型**

- 全量微调：更新所有参数，成本高，效果好
- 参数高效微调（PEFT）：只更新少量参数，成本低
- LoRA：PEFT的一种，最流行

**任务2：准备微调数据**

准备10-20条问答对：
```json
[
  {"instruction": "...", "input": "...", "output": "..."},
  ...
]
```

数据质量比数量更重要。

**任务3：用LoRA微调**

用LLaMA-Factory（最简单的工具）：
1. 下载模型（如Qwen2-7B）
2. 准备数据（JSON格式）
3. 配置LoRA参数
4. 开始训练

训练完成后，你就有一个"专属于你"的模型了。

### 今天的认知更新

- 微调不是必须的，先试Prompt Engineering和RAG
- LoRA是最实用的微调方法

## Day 6：部署与成本优化

### 目标

会用还不够，还要知道怎么部署和控制成本。

### 任务

**任务1：本地部署**

用Ollama（最简单的本地部署工具）：
```bash
ollama pull qwen2:7b
ollama run qwen2:7b
```

你的电脑上现在就跑着一个大模型了。

**任务2：API成本计算**

主流API的定价（2025年）：
- DeepSeek：1元/百万token（便宜）
- 通义千问：有免费额度
- GPT-4：约60元/百万token（贵）

计算你的使用场景的成本。比如：每天100次调用，每次1000个token，一个月多少钱？

### 今天的认知更新

- 本地部署适合隐私敏感场景
- API成本随使用量线性增长，需要监控
