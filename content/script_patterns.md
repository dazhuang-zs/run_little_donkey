## ⚠️ Pattern 是 toolbox 不是 mold（核心 meta 原则）

任何 pattern 都是**从你已发样本反推的"路径之一"**——不是默认应该 apply 的模板。

写稿前**永远先回答**：这篇稿子的最强属性是什么？再决定用什么 pattern。

## 通用 pattern 库（**初始抽象骨架，复盘后填具体**）

### Pattern 1：开头钩子（前 3 秒）

**原则**：绝不用"近年来 / 大家好 / 今天聊一聊"。

**已知有效的钩子类型**：
- **A. 第二人称场景代入 → 反转**
- **B. 戏仿原文体的影响声明**
- **C. 直接甩数据 / 反转性观察**

---

### Pattern 2：主体段落数量 + 切分

**原则**：避免 4-7 个并列概念。一般压缩到 **3 个**。

---

### Pattern 3：emotional 标记节点

**原则**：第一人称"我"出现 0-2 次，**永远在 emotional 高潮处**。

---

### Pattern 4：双声道结构（作者 + 用户旁白）

**原则**：
- 作者的话引出**框架**
- 你的话做**接地翻译**
- 你的话做**emotional 标记**

---

### Pattern 5：致谢段 / MVP 句不是必选

**原则**：只有当致谢段或 outro **本身就是 MVP 句**时才完整引用。

---

### Pattern 6：句子长度 / 段落节奏

**原则**：短句为主，避免复合句。

---

### Pattern 7：词汇风格

**原则**：建立你自己的词汇库——这是品牌识别的关键。

**避免**：
- 论文体连接词："其次" / "本研究" / "综上所述"
- 形容词堆砌
- 冗长定语
